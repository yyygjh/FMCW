## **边坡智能监测 Agent 系统**
端侧 AI 驱动边坡安全监测・本地离线自进化智能哨兵
纯本地离线可运行的边坡监测端侧大模型 Agent 系统，基于多传感器融合感知、8 级流水线拦截决策、双层记忆与自适应 RAG，实现感知→记忆→决策→执行全链路闭环，支持 7×24 小时野外无人值守边坡滑坡主动预警。

## 📌 项目概述

本系统面向山区、公路、矿山野外边坡监测场景，完全本地化部署，断网离线正常工作。
硬件层接入超声波位移传感器、北斗 GPS 定位模块、摄像头视觉设备实时采集边坡物理数据；本地轻量大模型 Agent 作为核心调度大脑，自主完成数据读取、趋势分析、风险诊断、主动告警、知识库迭代。
核心设计准则
零幻觉硬数据机制：位移、坐标、海拔、设备状态等数值 100% 由硬件工具接口返回，大模型不参与数值生成，彻底杜绝 AI 编造虚假监测数据；
主动智能而非被动问答：内置定时巡检线程，无需人工提问即可自动识别边坡位移突变、人员入侵，主动推送微信 / 邮件告警；
自进化闭环架构：每次边坡告警事件自动抽取特征存入向量知识库，系统使用越久诊断精度越高；
轻量化端侧适配：全部模型 CPU 可跑，无云端依赖，适配野外低功耗工控机、边缘盒子。

```language
│  - Serial串口：北斗模块备用数据通道，断网兜底数据源          │
│  - YOLOv8-n：摄像头实时画面，边坡区域人员/异物入侵视觉检测    │
└─────────────────┬───────────────────────────────────────────┘
                  │ 实时写入全局内存变量
┌─────────────────▼───────────────────────────────────────────┐
│  🧠 Agent决策核心层 SimpleAgent                              │
│  ┌─────────────────────────────────────────────────────┐    │
│  │ 8级优先级流水线拦截器（核心创新点）                  │    │
│  │ ①复合拆分 → ②事实提取 → ③用户笔记记忆拦截           │    │
│  │ ④天气/指代歧义拦截 → ⑤硬数据强制工具调用（防幻觉）  │    │
│  │ ⑥历史记忆检索 → ⑦ReAct自主推理 → ⑧LLM兜底应答       │    │
│  └─────────────────────────────────────────────────────┘    │
│  - 双层记忆体系：短期滑动对话记忆 + FAISS向量长期记忆       │
│  - 7组内置工具集：测距、GPS定位、天气查询、边坡知识库、滑坡诊断、传感器自检、历史事件检索
│  - 自进化闭环：风险预警触发 → 检索相似历史案例 → 自动写入本地知识库
└─────────────────┬───────────────────────────────────────────┘
                  │ 持久化读写
┌─────────────────▼───────────────────────────────────────────┐
│  💾 持久化数据存储层                                        │
│  - chat_histories/：多轮对话短期记忆（JSON文件存储）        │
│  - memory.json：用户配置、用户画像、长期对话摘要             │
│  - slope_memory.db：SQLite数据库，存储全量历史监测告警事件  │

└─────────────────────────────────────────────────────────────┘
```

⚡ 核心创新：8 级流水线拦截器
按优先级从高到低执行，上层拦截命中后直接终止后续流程，保证数据安全与响应速度
表格

| 优先级| 拦截模块|触发条件 |核心作用 |
| - | - | - | - |
| 1|复合意图拆分 |用户提问包含多意图、多标点复合问题 | 并行拆解多任务，分开处理|
| 2| 事实信息提取|包含「我叫、我的、阈值设为」等身份 / 配置语句 |自动提取用户基础信息存入记忆 |
|3 |用户笔记记忆拦截 |指令带「记住：」关键词 |主动留存自定义规则、边坡参数 |
|4 |天气 / 指代歧义拦截 |提问含今天天气、它、那边、此地等模糊指代 |消除上下文歧义，统一指代映射 |
|5 |硬数据强制工具调用 |问包含距离、位移、GPS、坐标、海拔、传感器数值 | 防幻觉核心，强制调用硬件工具，禁止模型生成数字|
|6 |历史记忆检索拦截|询问告警记录、历史数据、之前记录 |向量 / 数据库精准召回历史信息 |
|7 |	ReAct 自主推理 | Agent	闲聊、基础查询以外专业边坡诊断类问题| 模型自主规划工具调用链路，分步推理风险|
|8 |LLM 兜底回退 | 以上全部拦截未命中，普通闲聊问答| 通用对话兜底，轻量化本地大模型直接应答|

## 🚀 快速部署启动指南
### 1. 环境依赖安装
```bash
# 一键安装全部python依赖库
pip install -r requirements.txt
```
### 2. 提前部署 Ollama 本地模型
官网下载安装 Ollama 并启动服务
```bash
ollama serve
# 拉取对话大模型
ollama pull qwen2.5:3b-instruct
# 拉取向量嵌入模型
ollama pull qwen3-embedding:4b
```
### 3. 启动传感器模拟程序（无真实硬件时使用）
模拟 ESP8266 定时推送边坡位移、北斗 GPS 定位数据，用于本地调试开发
```bash
python sensor_simulator.py
```
### 4. 启动边坡监测主程序
```bash
python end_new.py
```
### 5. 系统 GUI 操作说明
实时监测面板：左侧可视化展示实时位移、GPS 坐标、采集时间、设备在线状态、位移趋势曲线
AI 智能对话框：右侧聊天窗口直接下发指令，示例提问：
北坡当前位移距离是多少？
边坡传感器设备运行是否正常？
近一小时有没有告警事件？
当前边坡滑坡风险等级判断
设备远程控制：界面按钮下发指令，控制现场 LED 指示灯、蜂鸣报警器
推理链路查看：点击「查看推理」，完整展示 8 级拦截器执行流程、工具调用日志
案例反馈优化：点击点赞 / 差评按钮收集 bad case，自动存入反馈文件夹用于迭代知识库
## 🧠 四大核心独创模块
#### 1. 硬数据防幻觉安全层（项目核心亮点）
强制拦截规则：所有涉及位移、GPS 坐标、海拔、传感器数值查询，直接跳转硬件工具读取真实数据
数据隔离：原始数值不传入 LLM，仅将工具返回结果整理后交给模型总结展示
异常重试机制：硬件通信失败自动重试 1 次，超时输出设备离线告警
#### 2. 双层分级记忆架构
短期记忆：FileChatMessageHistory + 5 轮滑动窗口，保存实时对话上下文；窗口满自动生成摘要写入长期存储，减少内存占用
长期记忆：FAISS 向量库为主、JSON 文件备份兜底，支持跨天、跨会话语义检索历史边坡记录、用户自定义参数
#### 3. 自适应动态 RAG 检索
根据用户问题类型自动切换检索参数，大幅提升问答准确率
事实参数类提问（阈值、规范、设备参数）：分块大小 200，检索 top-k=2
原理诊断类提问（滑坡成因、风险分析、处置方案）：分块大小 500，检索 top-k=4
迭代优化效果：基础检索准确率 24% → 自适应动态检索 94%（50 道边坡专业测试集验证）
#### 4. 自进化知识库闭环（独有创新）
后台常驻巡检线程：每 30 秒自动读取边坡位移时序数据，计算位移变化斜率判断风险
触发预警后自动执行：检索 FAISS 中相似历史滑坡案例 → 提取本次告警位移、时间、风险特征
自动写入知识库 rag_docs/slope_cases.txt，无需人工录入，系统持续自我完善

## 🛠️ 当前技术限制与未来演进路线
### 已知待优化问题
自然语言指代解析不完善：模糊代词「那边、此地、该边坡」天气匹配存在少量匹配失败场景，已定位拦截逻辑问题；
时序历史数据缓存 dist_history_with_time 未完全生效，数据量少时查询无报错但展示简略；
单 Agent 架构限制：多边坡分区、多设备并行任务处理能力较弱。
### 后续迭代演进方向
多 Agent 协同架构：新增 Supervisor 调度 Agent，拆分监测、告警、诊断、运维子 Agent 并行处理；
MCP 标准化工具协议：所有硬件、查询工具封装为 MCP Server，实现工具插拔式扩展；
全链路可观测平台：接入 LangSmith/LangFuse，可视化 Agent 推理、工具调用、RAG 检索全流程日志；
容器化服务改造：基于 FastAPI 封装接口，Docker 打包部署，支持多边缘节点分布式监测。
