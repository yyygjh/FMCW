【常见故障及解决】

1. 上位机连接不上 ESP8266 热点：
   - 检查电脑 WiFi 列表中是否存在 ESP8266_AP
   - 默认密码 12345678
   - 若看不到热点，重启 ESP8266 模块或重新烧录 AP 模式固件

2. 能连上热点但收不到距离数据：
   - 确认 C8T6 代码中 Wi-Fi 模式为 AP，且 TCP 端口为 8080
   - 使用网络调试助手尝试连接 192.168.4.1:8080 并发送 GET_DIST\n
   - 检查 Serial 串口是否输出原始距离值

3. 北斗串口无数据：
   - 确认硬件连接（TX→RX，GND→GND）
   - 检查设备管理器串口号是否为 COM5，波特率 115200
   - 可在终端用串口工具测试，看是否有 $GNGGA 等 NMEA 语句

4. YOLOv8 检测不到人或动物：
   - 确认摄像头已正确驱动（笔记本摄像头或 USB 摄像头）
   - 检查 yolov8n.pt 模型文件是否在根目录
   - 降低置信度参数 CONF_THRESH（默认 0.45）

5. 微信告警失败：
   - 程序启动后会自动弹出微信登录二维码，需扫码登录
   - 检查 itchat 库是否正常安装
   - 重启程序重新登录

6. 摄像头图像卡顿或黑屏，可能原因及解决：
   - 检查USB接口是否松动或供电不足（尝试换一个USB口或使用带供电的USB HUB）。
   - 摄像头驱动是否正确安装（设备管理器中是否有未知设备）。
   - 系统中是否有其他程序正在占用摄像头（关闭其他调用摄像头的应用）。
   - 降低图像分辨率或帧率（例如从640x480降到320x240）。
   - 检查树莓派或上位机的CPU负载是否过高（摄像头处理可能消耗资源）。

7. 雷达数据突然全部为 -1.0 的含义及处理：
   - -1.0 表示传感器未检测到有效回波。常见原因：
     * 雷达前方有障碍物或传感器被遮挡。
     * 测量距离超出有效量程（例如超过2米）。
     * 模块未正确初始化或通信协议错乱。
   - 建议：清理雷达前方区域，重启模块，检查串口/网络连接。

8. Ollama 模型调用超时的处理：
   - 可能原因：模型加载慢、系统资源不足、进程阻塞。
   - 解决方案：
     * 增加超时时间（如从20秒改为30秒或更长）。
     * 确保模型已提前加载（运行一次 `ollama run qwen2.5:0.5b-instruct` 预热）。
     * 检查系统内存和CPU，关闭其他占用资源的程序。
     * 如果频繁超时，考虑使用更小的模型（如 qwen2.5:0.5b）。

9. 上位机提示“WiFi连接断开”后的自动重连机制：
   - TCP线程会检测到连接异常后，每隔3秒尝试重新连接服务器（IP:192.168.0.48:8080），直至成功。
   - 日志中会输出“WiFi连接断开，尝试重连...”提示。

10. 北斗串口数据出现乱码，可能原因及解决：
   - 检查波特率是否匹配（应与程序中一致，默认115200）。
   - 检查串口连接线是否接触不良、过长或受干扰。
   - 确认接收的数据格式是否正确（NMEA 0183 协议），可尝试换用串口助手测试。
   - 如果乱码偶发，可增加数据校验或重启串口线程。
