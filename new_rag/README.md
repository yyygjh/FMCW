边坡智能监测 Agent 系统
端侧 AI 驱动的边坡安全监测系统 · 自进化 Agent 闭环 · 7×24 小时无人值守

📌 项目概述
本项目是一套纯本地、可离线运行的端侧 AI Agent 系统，专为野外边坡安全监测设计。系统通过超声波传感器、北斗 GPS 和摄像头实时感知物理世界，由本地大模型 Agent 自主决策，实现从 “感知 → 记忆 → 决策 → 执行” 的完整闭环。

核心理念：硬数据 100% 由工具返回，杜绝模型编造；Agent 从被动问答进化为主动预警、自学习、自进化的值班哨兵。

8 级拦截器流水线
优先级	拦截器	触发条件	目的
1	复合拆分	含多个标点/意图	并行处理多问题
2	事实提取	“我叫/我是”	记住用户偏好
3	用户笔记	“记住：”	主动记忆
4	天气/指代拦截	“今天天气”/“它”	高频歧义处理
5	强制工具调用	距离/GPS/天气关键词	防幻觉核心
6	历史/记忆查询	“告警”/“我叫什么”	精确检索
7	ReAct Agent	非闲聊未被拦截	模型自主决策
8	回退 LLM	全部未命中	通用兜底

🚀 快速开始
1. 环境准备
bash
# 安装依赖
pip install -r requirements.txt

# 启动 Ollama（需提前安装）
ollama serve
ollama pull qwen2.5:3b-instruct
ollama pull qwen3-embedding:4b
2. 启动模拟程序（感知层模拟器）
bash
python sensor_simulator.py  # 模拟 ESP8266 发送距离 + GPS 数据
3. 启动上位机
bash
python end_new.py
4. 使用方式
实时监测：左侧面板查看距离/GPS/时间实时数据

AI 对话：右侧聊天框直接提问（如“北坡距离多少？”“今天天气怎么样？”“传感器正常吗？”）

远程控制：点击 LED/蜂鸣器按钮下发控制指令

推理过程：点击“查看推理”按钮查看 Agent 决策链路

反馈优化：点击“👍”按钮收集 bad case

🧠 四大核心模块
1. 硬数据防幻觉层
强制拦截：所有含“距离/GPS/坐标/海拔”关键词的查询，直接调用工具返回

100% 可靠性：硬数据不经过 LLM，从源头杜绝编造

工具重试机制：调用失败自动重试 1 次

2. 双层记忆架构
短期记忆：FileChatMessageHistory + 滑动窗口（最近 5 轮）

摘要压缩：截断前异步生成摘要存入长期记忆

长期记忆：FAISS 向量存储 + JSON 回退，支持语义检索

3. 自适应 RAG 检索
事实类问题（“阈值是多少”）：chunk_size=200, k=2

原理类问题（“为什么能测距”）：chunk_size=500, k=4

优化路径：24% → 94%（50 题测试集）

4. 自进化闭环（项目独有）
触发：自主监控线程每 30 秒巡检距离趋势

进化：预警触发时自动检索历史案例 → 写入 rag_docs/cases.txt

效果：知识库持续生长，越用越好用

🛠️ 技术债务与演进方向
已知限制
天气解析：代词映射未完全覆盖（“那边会”“这个天气”仍可能失败）——已定位

时间上下文：dist_history_with_time 缓存未生效——数据不足时不报错

单 Agent 架构：适合端侧场景，但无法处理多任务并行

未来演进
多 Agent 协作：引入 Supervisor Agent 负责任务拆解

MCP 协议接入：将工具封装为标准化 MCP Server

可观测性升级：接入 LangSmith/LangFuse 可视化追踪

服务化部署：FastAPI + Docker 容器化
