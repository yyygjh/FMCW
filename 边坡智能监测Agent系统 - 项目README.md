# 边坡智能监测Agent系统 \- 项目README

# 🌿 边坡智能监测 Agent 系统

**端侧AI驱动边坡安全监测 · 本地离线自进化智能哨兵**

纯本地离线可运行的边坡监测端侧大模型Agent系统，基于多传感器融合感知、8级流水线拦截决策、双层记忆与自适应RAG技术，构建**感知→记忆→决策→执行**全链路闭环，支持7×24小时野外无人值守边坡滑坡主动预警，彻底摆脱云端依赖，适配野外断网恶劣工况。

## 📌 项目完整介绍（软硬件一体化系统）

本项目是一套**嵌入式端侧感知 \+ 本地Agent智能决策**的全自研、纯离线边坡安全智能监测系统，完整覆盖底层硬件搭建、嵌入式固件开发、物联网通信调试、视觉算法部署、上位机软件开发、RAG知识库搭建与智能Agent调度全链路，是一套真正落地、可工程化部署的**端边协同无人值守监测系统**。

系统分为**嵌入式感知下位机**与**AI智能决策上位机**两大核心部分：下位机基于双STM32架构完成全硬件采集、外设驱动、无线通信与数据预处理，解决传统嵌入式设备外设冲突、通信不稳定、野外适配性差等问题；上位机依托轻量化本地大模型Agent，融合自适应RAG、双层记忆架构、八级拦截决策机制，实现数据可视化、风险主动预警、设备智能运维、系统自进化迭代，全程无云端依赖，适配山区、矿山、基坑等断网野外复杂工况。

### 核心设计准则

- **零幻觉硬数据机制**：位移、坐标、海拔、设备状态等核心数值全部由硬件工具接口真实返回，大模型不参与数值生成，从根源杜绝AI编造虚假监测数据，保障工业监测可靠性

- **主动智能而非被动问答**：内置定时巡检后台线程，无需人工提问，自动识别边坡位移突变、人员入侵、设备异常，主动推送微信/邮件告警信息

- **自进化闭环架构**：系统每次监测告警事件自动提取核心特征，存入向量知识库，实现使用越久、故障案例越多、风险诊断精度越高的自迭代效果

- **轻量化端侧适配**：全部模型支持CPU离线运行，无云端服务器依赖，完美适配野外低功耗工控机、边缘盒子等轻量化硬件设备

## 🏗️ 整体系统完整架构（软硬件全链路）

系统采用**嵌入式感知层 \+ Agent决策层 \+ 数据持久层**三级解耦架构，打通「硬件采集→嵌入式固件处理→物联网传输→AI智能分析→决策预警→自主迭代」完整闭环，软硬件深度适配，针对性优化野外边坡监测工程痛点。

### 1\. 嵌入式感知下位机层（核心硬件底座）

自研嵌入式硬件监测平台，创新性采用**双STM32主控解耦架构**，彻底解决单单片机外设冗余、引脚冲突、代码耦合、编译报错等问题，实现功能模块化拆分、各司其职，大幅提升系统稳定性与可拓展性。完成全流程硬件选型、焊接调试、固件烧录与外设适配，攻克多项嵌入式工程疑难问题。

- **主控架构优化**：采用双STM32分工架构，分别负责传感器数据采集、外设控制与数据通信、指令交互，实现功能解耦，规避单开发板资源抢占、程序卡死、代码冲突问题

- **WiFi通信工程化迭代**：深度调试ESP8266无线模块，解决固件不兼容、烧录接触不良、模块无响应等硬件问题；创新性将通信模式从传统STA动态IP模式迭代为**AP固定热点模式**，彻底解决野外调试IP动态变换、电脑断网、通信抖动、连接不稳定等核心痛点，保障野外无人值守稳定传输

- **多传感器驱动开发**：独立编写底层驱动，完成超声波测距模块、北斗GPS定位模块、报警蜂鸣器、状态指示灯全套外设开发，实现边坡位移精准采集、地理位置实时定位、超限自动报警、设备状态自检功能

- **北斗/GPS适配优化**：解决定位模块搜星失败、室内无数据、串口乱码、接线报错、供电不稳等调试难题，优化串口8N1通信配置、硬件共地、射频天线适配、稳压供电方案，实现全天候稳定定位数据输出

- **视觉算法底层适配**：完成YOLOv8\-n目标检测、Allweather\-Net图像增强算法PC端调试与适配，优化雨、雾、暗光复杂场景图像识别效果，打通视觉监测与下位机传感数据联动逻辑，实现边坡人员入侵、异物遮挡智能识别

- **底层逻辑闭环**：基于C语言完成下位机全套逻辑开发，实现数据采集、阈值判断、异常报警、数据滤波、串口解析、WiFi透传全流程稳定运行，具备高容错、高稳定性的嵌入式工程特性

下位机采集的位移、定位、设备状态、视觉识别数据，通过WiFi TCP/串口双链路实时上传至上位机，为Agent智能决策提供真实、可靠、完整的硬件原始数据支撑。

### 2\. Agent决策核心层（SimpleAgent）

系统核心智能调度大脑，依托**8级优先级流水线拦截器、双层记忆体系、多功能工具集、自进化机制**，实现全场景智能决策。

#### ✨ 核心创新：8级优先级流水线拦截器

按优先级从高到低逐级执行，上层规则命中后直接终止后续流程，兼顾**数据安全性、响应速度、推理精准度**，彻底解决大模型幻觉、语义歧义、指令处理混乱问题。

|优先级|拦截模块|触发条件|核心作用|
|---|---|---|---|
|1|复合意图拆分|用户提问包含多意图、多标点复合问题|自动并行拆解多任务，分模块独立处理，避免指令混淆|
|2|事实信息提取|包含身份、设备阈值、自定义配置等语句|自动提取用户配置信息，持久化存入用户记忆|
|3|用户笔记记忆拦截|指令携带「记住：」自定义关键词|主动留存用户自定义边坡参数、运维规则|
|4|天气/指代歧义拦截|提问含模糊代词、地域、天气相关模糊指令|消除上下文语义歧义，统一指代映射标准|
|5|硬数据强制工具调用|查询位移、GPS坐标、海拔、传感器数值等硬件数据|防幻觉核心机制，强制调用硬件工具，禁止模型生成虚假数值|
|6|历史记忆检索拦截|查询历史告警、时序数据、过往监测记录|通过向量库\+数据库精准召回历史工况与案例|
|7|ReAct自主推理|边坡风险诊断、故障分析、方案咨询等专业问题|模型自主规划工具调用链路，分步推理、精准研判风险|
|8|LLM兜底应答|以上规则均未命中的普通闲聊问答|轻量化本地大模型直接应答，保障对话完整性|

#### 双层记忆体系

- **短期滑动记忆**：基于文件会话记忆，保留5轮实时对话上下文，窗口满自动生成摘要，减少内存占用，保障实时交互流畅度

- **长期向量记忆**：以FAISS向量数据库为核心、JSON文件备份兜底，支持跨会话、跨天数语义检索历史监测数据、告警案例、自定义参数

#### 7组内置工具集

集成测距采集、GPS定位、天气查询、边坡知识库检索、滑坡风险诊断、传感器自检、历史事件检索七大工具，覆盖边坡监测全场景业务需求。

#### 自进化闭环机制

系统触发风险预警后，自动检索向量库相似历史案例，提取本次告警的位移数据、时间节点、风险特征，自动写入本地知识库，无需人工录入，实现持续自我迭代优化。

### 3\. 持久化数据存储层

分级存储会话数据、设备配置、时序监测数据、告警案例、故障手册，实现数据全生命周期可追溯。

- `chat_histories/`：多轮对话短期记忆，JSON格式分会话存储

- `memory.json`：存储用户画像、自定义配置、长期对话摘要

- `slope_memory.db`：SQLite数据库，持久化全量时序监测数据、边坡告警事件

- `rag_docs/`：RAG专属知识库，包含边坡历史案例、设备故障手册、行业安全监测规范

## 🧠 四大核心独创模块

### 1\. 硬数据防幻觉安全层（核心亮点）

针对工业监测数据真实性痛点设计专属安全机制，所有数值类查询强制触发硬件工具调用，原始硬件数据不直接传入大模型，仅将工具校验后的标准化结果交由模型整理输出；同时配置通信异常自动重试机制，超时判定设备离线并推送告警，100%保障监测数据真实可靠。

### 2\. 双层分级记忆架构

兼顾实时交互效率与长期数据检索能力，短期滑动窗口保障多轮对话连贯性，满容量自动摘要归档；长期向量记忆实现跨会话语义检索，精准匹配历史边坡工况与故障案例，解决传统单记忆架构内存冗余、检索精度低的问题。

### 3\. 自适应动态RAG检索

根据用户提问场景自动切换检索参数，实现精准适配：参数配置类问题采用小分块、少检索量，提升响应速度；故障诊断、风险分析类问题采用大分块、多检索量，保障信息完整性。经50道边坡专业测试集验证，问答准确率从基础检索24%提升至94%。

### 4\. 自进化知识库闭环

后台常驻30秒周期巡检线程，实时分析边坡位移时序数据与变化斜率，自动判定风险等级；触发告警后主动检索相似历史案例，提取本次告警核心特征，自动录入知识库，无需人工干预，实现系统自主学习、持续迭代优化。

## 🚀 快速部署启动指南

### 1\. 环境依赖安装

```Plain Text
pip install -r requirements.txt
```

### 2\. 本地大模型部署（Ollama）

```Plain Text
# 启动Ollama本地服务
ollama serve
# 拉取对话推理模型
ollama pull qwen2.5:3b-instruct
# 拉取向量嵌入模型
ollama pull qwen3-embedding:4b
```

### 3\. 启动传感器模拟程序（无硬件调试专用）

```Plain Text
python sensor_simulator.py
```

模拟ESP8266、北斗设备定时推送边坡位移、定位数据，满足本地开发调试需求。

### 4\. 启动系统主程序

```Plain Text
python end_new.py
```

- 实时监测面板：可视化展示实时位移、GPS坐标、采集时间、设备在线状态、位移趋势曲线

- 设备控制：一键控制现场LED指示灯、蜂鸣报警器

- 推理链路可视化：完整展示8级拦截器执行流程、工具调用日志

## 🛠️ 技术局限与未来迭代路线

### 当前已知待优化问题

- 自然语言指代解析不完善，少量模糊代词、地域指代存在匹配失败场景
